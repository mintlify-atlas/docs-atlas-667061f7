Get shvm-db running locally in under 5 minutes. This guide walks you through installation, local development, and your first DynamoDB-compatible operations.

## Prerequisites

Before you begin, ensure you have:

- Node.js 18 or later
- npm 10.9.2 or later (specified in package.json)
- A Cloudflare account (for deployment)
- Git (to clone the repository)

## Installation

<Steps>
  <Step title="Clone the repository">
    ```bash
    git clone https://github.com/shivamd20/shvm-db.git
    cd shvm-db
    ```
  </Step>

  <Step title="Install dependencies">
    ```bash
    npm install
    ```

    This installs all required dependencies including:
    - `wrangler` - Cloudflare Workers CLI
    - `@cloudflare/workers-types` - TypeScript definitions
    - `vitest` - Testing framework
    - `zod` - Schema validation
  </Step>

  <Step title="Start the development server">
    ```bash
    npm run dev
    ```

    Or alternatively:
    ```bash
    npm start
    ```

    Both commands run `wrangler dev`, which starts a local development server with:
    - Hot reload on file changes
    - Local Durable Objects emulation
    - SQLite database per partition
  </Step>
</Steps>

Your shvm-db instance is now running at `http://localhost:8787`!

## Your First Request

Let's create a table and put an item using the DynamoDB-compatible API.

### Create a Table

```bash
curl -X POST http://localhost:8787/api \
  -H "Content-Type: application/x-amz-json-1.0" \
  -H "x-amz-target: DynamoDB_20120810.CreateTable" \
  -d '{
    "TableName": "Users",
    "KeySchema": [
      { "AttributeName": "userId", "KeyType": "HASH" }
    ],
    "AttributeDefinitions": [
      { "AttributeName": "userId", "AttributeType": "S" }
    ]
  }'
```

### Put an Item

```bash
curl -X POST http://localhost:8787/api \
  -H "Content-Type: application/x-amz-json-1.0" \
  -H "x-amz-target: DynamoDB_20120810.PutItem" \
  -d '{
    "TableName": "Users",
    "Item": {
      "userId": { "S": "user123" },
      "name": { "S": "Alice" },
      "email": { "S": "alice@example.com" }
    }
  }'
```

### Get an Item

```bash
curl -X POST http://localhost:8787/api \
  -H "Content-Type: application/x-amz-json-1.0" \
  -H "x-amz-target: DynamoDB_20120810.GetItem" \
  -d '{
    "TableName": "Users",
    "Key": {
      "userId": { "S": "user123" }
    }
  }'
```

Response:
```json
{
  "Item": {
    "userId": { "S": "user123" },
    "name": { "S": "Alice" },
    "email": { "S": "alice@example.com" }
  }
}
```

## Understanding the Architecture

shvm-db uses a **deterministic routing architecture**:

1. **TableRegistryDO**: Global registry storing metadata for all tables
2. **PartitionDO**: Maps to `TableName + PartitionKey` hash, owns one SQLite database
3. **Edge Cache**: Cloudflare Cache API for fast reads (5-minute TTL for items)

```text
Partition Key → SHA-256 Hash → PartitionDO (Storage Executor)
```

Each PartitionDO:
- Runs a single SQLite database in WAL mode
- Guarantees strong per-partition consistency
- Provides single-writer semantics per partition key

## What's Supported

shvm-db implements core DynamoDB operations:

| Operation | Status | Notes |
|-----------|--------|-------|
| `CreateTable` | ✅ Yes | Instantly active |
| `DeleteTable` | ✅ Yes | Removes metadata |
| `ListTables` | ✅ Yes | Supports pagination |
| `DescribeTable` | ✅ Yes | Returns schema metadata |
| `PutItem` | ✅ Yes | Full condition expressions |
| `GetItem` | ✅ Yes | Consistent reads |
| `UpdateItem` | ✅ Yes | Partial support for UpdateExpression |
| `DeleteItem` | ✅ Yes | Full condition expressions |
| `Query` | ❌ No | Planned |
| `Scan` | ❌ No | Excluded from MVP |
